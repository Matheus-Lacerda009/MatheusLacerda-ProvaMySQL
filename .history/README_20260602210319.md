# Sistema de Gerenciamento Industrial - Banco de Dados Relacional

## 📋 Sobre o Projeto

Este projeto foi desenvolvido como atividade prática da Unidade Curricular **Implantação de Banco de Dados** do **SENAI Santa Catarina**.

O objetivo é implementar uma solução completa de banco de dados relacional capaz de armazenar, organizar e disponibilizar informações de uma indústria, permitindo o gerenciamento de:

* Setores da empresa;
* Funcionários;
* Categorias de produtos;
* Fornecedores;
* Produtos;
* Ordens de produção.

O projeto contempla modelagem de dados, implementação física do banco, inserção de dados para testes e consultas SQL para análise das informações.

---

## 🎯 Objetivos

* Aplicar técnicas de modelagem de dados.
* Implementar banco de dados relacional utilizando SQL.
* Utilizar chaves primárias e estrangeiras.
* Garantir integridade referencial entre tabelas.
* Executar consultas para recuperação e análise de dados.
* Implantar o ambiente utilizando Docker Compose.

## 📂 Estrutura do Banco de Dados

### Tabelas

#### Setores

Armazena os setores da empresa.

| Campo       | Tipo         |
| ----------- | ------------ |
| id_setor    | INT          |
| nome        | VARCHAR(100) |
| localizacao | VARCHAR(255) |

---

#### Funcionarios

Armazena informações dos funcionários.

| Campo          | Tipo         |
| -------------- | ------------ |
| id_funcionario | INT          |
| nome           | VARCHAR(100) |
| cpf            | VARCHAR(11)  |
| cargo          | VARCHAR(100) |
| salario        | DECIMAL(8,2) |
| data_admissao  | DATE         |
| id_setor       | INT (FK)     |

---

#### Categoria_Produto

Armazena categorias dos produtos.

| Campo          | Tipo         |
| -------------- | ------------ |
| id_categoria   | INT          |
| tipo_categoria | VARCHAR(100) |

---

#### Fornecedores

Armazena fornecedores da empresa.

| Campo         | Tipo         |
| ------------- | ------------ |
| id_fornecedor | INT          |
| cnpj          | VARCHAR(14)  |
| telefone      | VARCHAR(11)  |
| cidade        | VARCHAR(100) |

---

#### Produtos

Armazena os produtos fabricados.

| Campo              | Tipo         |
| ------------------ | ------------ |
| id_produto         | INT          |
| nome               | VARCHAR(100) |
| descricao          | TEXT         |
| preco_fabricacao   | DECIMAL(5,2) |
| quantidade_estoque | INT          |
| codigo             | VARCHAR(30)  |
| id_categoria       | INT (FK)     |
| id_fornecedor      | INT (FK)     |

---

#### Ordem_Producao

Controla as ordens de produção.

| Campo           | Tipo     |
| --------------- | -------- |
| id_ordem        | INT      |
| data_prod       | DATE     |
| qtd_prod        | INT      |
| status_prod     | ENUM     |
| tempo_estimado  | INT      |
| tempo_real_prod | INT      |
| id_produto      | INT (FK) |
| id_funcionario  | INT (FK) |

---

## 🔗 Relacionamentos

* Um setor possui vários funcionários.
* Um funcionário pertence a um único setor.
* Uma categoria pode possuir vários produtos.
* Um fornecedor pode fornecer vários produtos.
* Um produto pode possuir várias ordens de produção.
* Um funcionário pode ser responsável por várias ordens de produção.
